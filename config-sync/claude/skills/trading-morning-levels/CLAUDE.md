# trading-morning-levels — Claude Setup Guide

Draws Daily / 4H / 1H Volume Profile + S/R + HVN levels on NQ, ES, GC (and their micros) every morning via TradingView Desktop. Runs as a macOS launchd job at 8:30 AM ET Mon–Fri. After drawing, generates clean PNG screenshots of NQ and ES and posts to Discord.

## Quick Setup (new machine)

```bash
git clone https://github.com/fificlawbot/trading-morning-levels.git
cd trading-morning-levels

# Add Discord webhooks to macOS Keychain (required)
security add-generic-password -U -s trading-morning-levels \
    -a DISCORD_LEVELS_WEBHOOK -w '<chart_post_webhook_url>' -T /usr/bin/security
security add-generic-password -U -s trading-morning-levels \
    -a DISCORD_STATUS_WEBHOOK -w '<status_post_webhook_url>' -T /usr/bin/security

# Install everything (symlinks repo as skill dir, generates plist, loads launchd)
bash install.sh
```

## What install.sh does

1. Symlinks `~/.claude/skills/trading-morning-levels` → the cloned repo. The
   repo is the single source of truth — edits take effect immediately, no
   copy step. Runtime artifacts (logs, levels_*.json, bars_*.json) get
   written into the repo and are .gitignored.
2. Verifies macOS Keychain has `DISCORD_LEVELS_WEBHOOK` and `DISCORD_STATUS_WEBHOOK`
   (service `trading-morning-levels`); prints the `security` command to add them if missing.
3. Generates `~/Library/LaunchAgents/com.trading.morninglevels.plist` with correct `$HOME` paths.
4. Loads the launchd agent (fires Mon–Fri at 8:30 AM).
5. Checks Python deps and tradingview-mcp.

Install these before running `install.sh`:

```bash
# Python deps

```

## TradingView Layout Requirement

The NewYork layout **must be restored** after install:

- Tab 0 (`b9YsTOVW`) = NQ1! on any timeframe (used for bar fetching)
- The "Levels" watchlist must exist with: `CME_MINI:NQ1!`, `CME_MINI:ES1!`, `CME_MINI:MNQ1!`, `CME_MINI:MES1!`, `COMEX:GC1!`, `COMEX_MINI:MGC1!`

Start TV with CDP enabled (required for the script to control it):
```bash
/Applications/TradingView.app/Contents/MacOS/TradingView --remote-debugging-port=9222 &
```

**This command must be run from a real terminal (Terminal.app / iTerm), not a Claude Code subprocess — Electron requires a GUI session to launch.**

## Secrets

`morning_levels.sh` resolves each webhook in this order, first hit wins:

1. **macOS Keychain** (preferred) — service `trading-morning-levels`, account = the env var name
2. `~/.trading-morning-levels.env` — plain-text fallback for first-run / recovery only

Plain-text `.env` files in the repo or in Google Drive are intentionally not consulted (and `.env` is gitignored) — secrets should never live in synced cloud storage or version control.

Required Keychain entries:
- `DISCORD_LEVELS_WEBHOOK` — chart screenshots (NQ/ES posts)
- `DISCORD_STATUS_WEBHOOK` — daily run summary

Add or rotate:
```bash
security add-generic-password -U -s trading-morning-levels \
    -a DISCORD_LEVELS_WEBHOOK -w '<new_url>' -T /usr/bin/security
```
The `-T /usr/bin/security` ACL lets `security` re-read the entry from launchd context without an unlock prompt. The launchd plist no longer carries the webhook value — `morning_levels.sh` fetches it itself, so rotating Keychain alone is enough.

Inspect:
```bash
security find-generic-password -s trading-morning-levels -a DISCORD_LEVELS_WEBHOOK -w
```

Delete:
```bash
security delete-generic-password -s trading-morning-levels -a DISCORD_LEVELS_WEBHOOK
```

```bash
  <current_price>
```

## Known Issues

- **White screen on TV restart**: `morning_levels.sh` retries once automatically (10s sleep + retry)
- **Wrong symbol on NQ tab**: script verifies tab 0 quote matches NQ1!, falls back to set_symbol
- **VP VAL/VAH missing**: fixed — VP levels never range-filtered (only S/R is)
- **CDP not accessible from subprocess**: TV must be launched from user terminal, not Claude Code subprocess (Electron needs GUI session)

## Algorithm Notes

VP replicates **kv4coins** Pine Script (`PUB;X0dKLxdLh46AZbqGl3Bgxb83fo9oWlDe`):
- 68% value area (not 70%), 200 price buckets
- Lookbacks: Daily=10 bars, 4H=12 bars, 1H=today's session (22:00 UTC prev day)
- Fallback if < 4 session bars: last 20 1H bars

Level priority for dedup (within 40pt): Daily VP=5 > 4H/Session VP=4 > HVN=3 > S/R drawn separately
